# Journal Template

The journal lives at `~/Documents/ikigai/journal.md`. It is **append-only** — never rewrite, never summarize over, never delete.

## Initial Template (write this when the journal doesn't exist)

```markdown
# Ikigai Journal

Started: YYYY-MM-DD

A record of what makes my life feel worth living — the objects and the feeling (ikigai-kan).
Not a life-purpose statement. Not a Venn diagram. A living log.

---

## Sessions

<!-- Each completed session appended here, most recent last. -->

---

## Daily Log

<!-- One-line or two-line notices, one per day, most recent last. -->

---

## Patterns

<!-- Written after Session 7, and refreshed at re-audits. -->

---

## Ikigai-9 Trajectory

<!-- Each administration of the scale recorded here in order. -->
```

On first run, create the file with that exact template, with today's date filled in.

---

## Session Entry Format

Append each session under `## Sessions` using this shape:

```markdown
### Session N — <Name> — YYYY-MM-DD

**Opening state (before session):** <one sentence; whatever mood / question they arrived with>

**Carry-exercise from last session:** <what was assigned + how it went. For Session 1, skip.>

**Session work:**

<The session's content. Preserve the user's own words in quotes wherever possible. Organize
under the subheadings the session specifies. Be concrete — the whole point is that in six
months the user should be able to reread this and remember what they felt, not just what
they said in abstract.>

**Carry-exercise assigned:** <one concrete thing, specifically described>

**Notes for next time:** <anything you-the-skill want to remember: a thread to pick up,
a promise the user made, an emotion that passed by without being fully landed>
```

---

## Daily Log Entry Format

Append each check-in under `## Daily Log`:

```markdown
### YYYY-MM-DD

<User's small ikigai in their own words. One or two sentences. Optionally, one sensory detail.>
```

Do not interpret. Do not synthesize. Do not follow-up-question more than once. This section exists to build a dense raw corpus of small ikigai the user will read months later and notice patterns in.

## Rules for Writing to the Journal

1. **Append, never rewrite.** Even if the user later contradicts themselves, keep the old entry. The contradiction is itself information.
2. **Preserve the user's words.** Quote them. Don't paraphrase unless you must for brevity.
3. **Don't interpret during logging.** Save interpretation for the Patterns entry and for explicit synthesis moments.
4. **Time-stamp everything.** YYYY-MM-DD. The chronology matters.
5. **If you catch yourself summarizing what the user felt, stop.** Ask them to put it into words and write *their* words.
6. **Never show the Venn diagram in the journal.** Even to debunk it. The journal is sacred space for the authentic practice; save the debunk for the conversation.
